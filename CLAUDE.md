# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

AI Workspace — a self-hosted productivity platform (CRM, email, calendar, projects, tasks, document library, AI assistant) running on a single Contabo VPS via Docker Compose. Zero paid SaaS dependencies.

## Build and Run Commands

```bash
# Full stack (production)
docker compose up -d                    # start all 16 services
docker compose up -d --build api web    # rebuild + restart specific services
docker compose logs api --tail 50       # check logs
docker ps                               # verify health

# Local development (no Docker)
cd apps/api && npm install && npm run dev      # Fastify API on :4000 (uses node --watch)
cd apps/web && npm install && npm run dev      # Next.js on :3000
cd apps/workers && npm install && npm start    # BullMQ workers

# API tests
cd apps/api && npm test                        # runs node --test test/*.test.js

# Web build
cd apps/web && npm run build
```

Set `NEXT_PUBLIC_API_URL=http://localhost:4000` in `apps/web/.env.local` for local dev.

## Migrations

SQL migrations live in `db/migrations/` with naming pattern `0001_description.sql`. They run automatically on API startup via `apps/api/src/lib/migrate.js`. Each migration runs in a transaction and is tracked in the `schema_migrations` table. To add a migration, create the next numbered `.sql` file — it applies on next API restart.

## Architecture

**16 Docker services** on one compose file: Caddy (reverse proxy + auto-TLS), Postgres 16 + pgvector, Redis 7, MinIO (S3), MeiliSearch, Ollama (local LLM), Whisper (STT), Piper (TTS), Radicale (CalDAV), API, Web, Workers, Portal, Uptime Kuma, Dozzle, plus a piper-http sidecar.

**Three app services share the same Postgres and Redis:**

- **`apps/api`** — Fastify 4, Node 20, ES modules. Entry: `src/server.js`. Auth via session cookies (magic link + passkeys). Every route module exports a Fastify plugin that receives `app`. Public routes (auth, health) register first; all others require auth via `app.decorate("requireAuth", requireAuth)` applied per-route. Rate limiting backed by Redis. Request validation via Zod.

- **`apps/web`** — Next.js 14 (App Router), React 18, Tailwind CSS, PWA with service worker. Build requires `NEXT_DISABLE_SWC_NATIVE=1` on this VPS (SWC SIGBUS on QEMU/KVM). `NEXT_PUBLIC_API_URL` is baked in at build time via Docker build args.

- **`apps/workers`** — BullMQ workers: IMAP sync (every 2 min), embedding generation (pgvector, 768-dim via nomic-embed-text), daily digest, reminder dispatcher (web push), CalDAV two-way sync (`src/caldav.js`). Shares crypto helpers with API for IMAP password decryption.

- **`apps/portal`** — Client portal, separate Fastify app with a restricted `portal_reader` DB role.

**Data flow for AI assistant:** User message → API loads thread history + pgvector similarity search → builds prompt with today's events/tasks/threads → Ollama (or Anthropic/Groq depending on `AI_PROVIDER`) streams response via SSE → tool calls validated and executed server-side → results streamed back. Voice: browser audio → Whisper → text chat flow → Piper → WAV playback.

**Networking:** Two Docker networks — `edge` (Caddy-facing services) and `internal` (DB, Redis, Ollama, workers). Caddy terminates TLS for all subdomains. The web app proxies `/api/*` to Fastify via Caddy (strip prefix, flush for SSE).

## Key Conventions

- **Language:** JavaScript (ES modules, Node 20+). The web app uses TypeScript. Do not convert the API or workers to TypeScript.
- **Database access:** Raw `pg` with parameterized queries via `apps/api/src/lib/db.js` (exports `db` pool and `query` helper). No ORM. Slow queries (>500ms) are auto-logged.
- **Auth:** Session cookie (`session`) → SHA-256 hash lookup in `sessions` table. Per-user encryption key derived from `JWT_SECRET` via HMAC for storing IMAP passwords (AES-256-GCM).
- **AI tools:** Defined in `apps/api/src/lib/assistant.js` — schema in `toolSchemas`, execution in `runTool` switch. Always scoped to `req.user.user_id`.
- **All tables have `user_id`** for row-level data isolation. Never accept `user_id` as API input.
- **UUIDs everywhere** (v4, generated by Postgres `uuid_generate_v4()`).
- **Postgres extensions:** uuid-ossp, pgcrypto, vector, pg_trgm.

## Known Pitfalls

- **SWC SIGBUS:** Next.js SWC binary crashes on this QEMU/KVM VPS. Always set `NEXT_DISABLE_SWC_NATIVE=1`.
- **BigInt serialization:** Postgres bigints come back as strings from `pg` — keep them as strings in API responses. `Number(bigint)` overflows at 2^53.
- **Timestamp precision:** Postgres TIMESTAMPTZ has microseconds, JS Date has milliseconds. Truncate to ms for comparisons.
- **Route ordering:** In Fastify route plugins, public endpoints must be defined before any `preHandler` auth hook. 401 errors usually mean ordering, not broken auth.
- **Docker hostnames:** Use `redis`, `postgres`, `ollama`, `minio`, `meili` inside containers. Use `localhost` only in local dev.
- **Docker UID:** Host `app` user is UID 1000. Containers writing to `/data/*` must match. Permission errors after restart = container wrote as root.
- **Contabo IPs are greylisted.** Do not propose self-hosted mail servers. Use IMAP/SMTP to existing providers.

## What Not to Do

- Do not add n8n (removed deliberately — BullMQ workers replace it).
- Do not propose OpenAI/Anthropic/Google as required dependencies. Ollama is default; Groq free tier is the optional fallback.
- Do not suggest monorepo tools (Nx, Turborepo). The three-app structure is intentional.
- Do not `docker system prune -af` without asking — real data on real volumes.
- Do not reset/recreate the Postgres volume unless explicitly requested.
- Do not install packages globally inside containers.

## Deployment Checklist

1. `docker compose config` — validate YAML before applying.
2. `docker compose up -d` — apply changes.
3. `docker ps` — verify all containers show healthy/running.
4. `docker logs <name> --tail 50` — check any unhealthy container.
5. Never report success until containers are confirmed healthy.
